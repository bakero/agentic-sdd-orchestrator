# v0.4 Verification Checklist

Use this checklist to verify the v0.4 agent, skill, and environment profiles release. Run from the orchestrator repository unless noted otherwise.

## Automated checks

- [ ] `npm test` passes.
- [ ] `npm run typecheck` passes.

## Configuration

- [ ] `npm run agentic-sdd -- config init` creates `.agentic-sdd/config.json`.
- [ ] `npm run agentic-sdd -- config init` (again, no `--force`) reports the file already exists and does not overwrite it.
- [ ] `npm run agentic-sdd -- config validate` passes with no errors.
- [ ] `npm run agentic-sdd -- config show` prints the effective config and correctly reports its source (local override vs. built-in defaults).

## Agents, profiles, environments

- [ ] `npm run agentic-sdd -- agent list` shows all 5 default agents with role, default profile, execution mode, and provider.
- [ ] `npm run agentic-sdd -- agent show codex_architect` shows description, external reference, skills, responsibilities, and forbidden actions.
- [ ] `npm run agentic-sdd -- profile list` shows all 5 default profiles.
- [ ] `npm run agentic-sdd -- profile show technical_specification` shows prompt template, context policy, recommended reasoning, and which agent uses it.
- [ ] `npm run agentic-sdd -- env list` shows all 5 default environments.
- [ ] `npm run agentic-sdd -- env show claude_cowork_browser` shows tools, execution rules, forbidden actions, and expected output format.

## Existing project registry / doctor / next commands still work

- [ ] `npm run agentic-sdd -- project list` still works unchanged.
- [ ] `npm run agentic-sdd -- doctor .` still works unchanged.
- [ ] `npm run agentic-sdd -- next .` still works unchanged (if exercised).
- [ ] `npm run agentic-sdd -- inspect .` still works unchanged.
- [ ] `npm run agentic-sdd -- project config <name-or-path>` shows agents/environments that would apply to the resolved project.

## Safety guarantees observed

- [ ] No autonomous agent execution occurred at any point.
- [ ] No external AI API call occurred at any point.
- [ ] No automatic merge or PR creation occurred at any point.
- [ ] `profile`, `agent`, `env`, `config show`, `config validate`, and `project config` did not modify any target repository or the orchestrator repository outside `.agentic-sdd/`.
- [ ] `config init` only ever wrote inside `.agentic-sdd/` in the orchestrator repo.

## Local user config handling

- [ ] `.agentic-sdd/config.json` is gitignored: `git status --short` does not list it after `config init` has run.
- [ ] Deleting/never creating `.agentic-sdd/config.json` still leaves every command working (falls back to built-in defaults).

## Backwards compatibility

- [ ] Existing v0.1/v0.2/v0.3 tests (`installer.test.ts`, `project-manager.test.ts`, pre-existing `index.test.ts` cases) still pass.
- [ ] `install`, `init-feature`, `project add/list/remove/inspect`, `doctor`, `next` are unchanged in behavior.
