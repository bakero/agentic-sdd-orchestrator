# v0.5 Verification Checklist

Use this checklist to verify the v0.5 multi-agent Cowork handoff release. Run from the orchestrator repository unless noted otherwise.

## Automated checks

- [ ] `npm test` passes.
- [ ] `npm run typecheck` passes.
- [ ] `npm run agentic-sdd -- config validate` passes.

## Backwards compatibility

- [ ] `npm run agentic-sdd -- project list` still works unchanged.
- [ ] `npm run agentic-sdd -- doctor <name>` still works unchanged.
- [ ] `npm run agentic-sdd -- next <name>` still works unchanged, plus now suggests `handoff generate` once a project is `READY`/`CHECK_WARNINGS`.
- [ ] `npm run agentic-sdd -- agent list` / `env list` still work unchanged.
- [ ] Existing v0.1/v0.2/v0.3/v0.4 tests (`installer.test.ts`, `project-manager.test.ts`, `config-profiles.test.ts`, pre-existing `index.test.ts` cases) still pass.

## Handoff generation

- [ ] `handoff generate <project>` fails clearly with the exact install command when the runtime kit is missing.
- [ ] `handoff generate <project>` fails clearly with the exact `init-feature` command when no feature exists.
- [ ] `handoff generate <project>` fails clearly and lists candidates when multiple features exist and `--feature` is not passed.
- [ ] `handoff generate <project>` auto-resolves a single existing feature.
- [ ] `handoff generate <project> --feature <id>` resolves the requested feature among several.
- [ ] `handoff generate <project> --agent <name>` overrides the recommended agent.
- [ ] `handoff generate <project> --environment <name>` overrides the environment.
- [ ] Generated prompt includes agent/profile/environment details (section E).
- [ ] Generated prompt includes the overall orchestration goal (section C) and this-step goal (section D).
- [ ] Generated prompt includes inputs (section F) and expected outputs (section G).
- [ ] Generated prompt includes allowed paths (section H) and forbidden actions (section I), including the fixed safety baseline (no auto-merge, no external AI APIs, no secrets, no deleting user files).
- [ ] Generated prompt includes validation commands (section J) and the final report format (section K).
- [ ] Generated prompt includes browser/Cowork-specific instructions when the environment is `claude_cowork_browser` or `manual_copy_paste`.
- [ ] `handoff generate` does not modify the target repository.

## Handoff write / show / list

- [ ] `handoff write <project>` creates `handoff.json`, `prompt.md`, and `context_files.txt` under `.agentic-sdd/handoffs/<project-slug>/<feature-id>/<timestamp>/`.
- [ ] `handoff write` does not write into the target repository.
- [ ] `handoff show <project>` reprints the most recently written handoff.
- [ ] `handoff show <project>` suggests `handoff write` when none exists yet.
- [ ] `handoff list <project>` lists previously written handoffs, newest first.

## Safety guarantees observed

- [ ] No autonomous agent execution occurred at any point.
- [ ] No external AI API call occurred at any point.
- [ ] No automatic merge or PR creation occurred at any point.
- [ ] `.agentic-sdd/handoffs/` is gitignored: `git status --short` does not list it after `handoff write` has run.
- [ ] Every generated prompt's forbidden-actions section includes the fixed safety baseline plus the specific agent's own forbidden actions.
