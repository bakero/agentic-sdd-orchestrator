# Agentic SDD Orchestrator v0.1 Demo Release

## Release status

This is a demo-ready local MVP for Cowork mode.

It is suitable for showing the end-to-end setup and first prompt-generation flow in a local target repository. It is not a commercial production release.

## What the product is

Agentic SDD Orchestrator prepares a repository for a human-supervised agent workflow.

In v0.1, the product:

- inspects a local target repository;
- installs the Agentic SDD runtime kit into that repository;
- initializes a first feature workflow folder and branch;
- validates workflow state;
- resolves the next handoff;
- renders `.agent_runtime/next_prompt.md` for manual or semi-assisted execution in Claude Cowork, Codex, Gemini, or similar tools.

## What v0.1 supports

- local filesystem target repositories;
- Cowork-mode prompt generation only;
- `inspect`, `install`, `install --dry-run`, and `init-feature`;
- runtime kit installation without overwriting existing files by default;
- feature folder scaffolding from the runtime template;
- `.agent_runtime/` gitignore enforcement;
- first prompt generation through `npm run agent:next`;
- human review and human-controlled merge.

## What v0.1 does not support

- dashboard;
- API mode;
- autonomous execution;
- automatic merge;
- external AI API calls;
- secrets or credential handling;
- GitHub App installation;
- remote repository cloning;
- published CLI packaging.

## Demo flow summary

The intended demo flow is:

1. Create a fresh local sandbox repository.
2. Run `npx tsx packages/cli/src/index.ts install <sandbox>` from the orchestrator repo.
3. Run `npx tsx packages/cli/src/index.ts init-feature <sandbox> --issue 1 --slug demo-feature --title "Demo feature"` from the orchestrator repo.
4. Inside the sandbox, run `npm install`.
5. Inside the sandbox, run `npm run agent:next`.
6. Open `.agent_runtime/next_prompt.md`.
7. Paste that prompt into Claude Cowork, Codex, Gemini, or a similar human-supervised tool.

## Safety guarantees

- no autonomous execution;
- no automatic merge;
- no hidden agent actions;
- no external AI API calls;
- GitHub and `status.md` remain the source of truth for installed repositories;
- prompt generation remains local and semi-assisted;
- human final merge remains required.
