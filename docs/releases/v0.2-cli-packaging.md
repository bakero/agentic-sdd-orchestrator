# v0.2 CLI Packaging

## Status

Technical release candidate.

## Summary

v0.2 prepares the repository for a cleaner first technical release by improving local CLI packaging, CLI usability, documentation, and verification without changing the v0.1 Cowork-mode safety model.

## What changed from v0.1

- added the recommended local command style: `npm run agentic-sdd -- ...`
- improved CLI usage and validation for missing commands, target repos, unknown commands, and required `init-feature` flags
- added focused CLI invocation tests
- expanded release and demo documentation around the local install flow

## Recommended command

```bash
npm run agentic-sdd -- inspect <target-repo>
npm run agentic-sdd -- install <target-repo>
npm run agentic-sdd -- init-feature <target-repo> --issue 1 --slug demo-feature --title "Demo feature"
```

## Supported flow

1. `npm run agentic-sdd -- install <target-repo>`
2. `npm run agentic-sdd -- init-feature <target-repo> --issue <number> --slug <slug> --title <title>`
3. `cd <target-repo>`
4. `npm install`
5. `npm run agent:next`
6. Open `.agent_runtime/next_prompt.md`

## Non-goals

- no dashboard
- no API mode
- no publishing
- no autonomous execution
- no auto-merge

## Known limitations

- local filesystem only
- not packaged to npm yet
- GitHub App not available
- no remote cloning
- no migration system

## Future-ready packaging note

The repository now exposes a clean local npm script entrypoint. A standalone `agentic-sdd` package bin is intentionally deferred until the CLI is built to JavaScript output, so the project does not pretend that raw TypeScript packaging is production-ready.
